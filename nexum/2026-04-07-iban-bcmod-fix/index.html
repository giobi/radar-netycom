Il problema: errore durante il pagamento con IBAN

Alcuni clienti, completando l’acquisto e scegliendo il pagamento tramite IBAN bancario, ricevevano un messaggio di errore e non riuscivano a concludere l’operazione. Il problema si verificava quando l’IBAN veniva inserito con spazi o trattini — un formato molto comune, perché è così che le banche lo mostrano negli estratti conto e nelle app.

Esempio: scrivere IT 60 X054 2811 1010 0000 0123 456 invece di IT60X0542811101000000123456 causava il blocco.

La schermata di scelta pagamento, dove il cliente seleziona SEPA (addebito bancario con IBAN)
Errore riprodotto in ambiente di test

Abbiamo riprodotto esattamente lo stesso errore segnalato nei log del 3 aprile, inserendo un IBAN con spazi nel campo dedicato. Cliccando “Controlla IBAN e procedi”, il sito ha restituito un errore 500 (pagina bianca con “Server Error”). Conferma che la causa è stata individuata correttamente.

IBAN inserito con spazi, come viene tipicamente copiato dall’app della banca
Risultato: errore 500 — il sito si blocca e il pagamento non va a buon fine
Correzione applicata e verificata

Ora il sistema pulisce automaticamente l’IBAN prima di controllarlo: rimuove spazi, trattini e qualsiasi carattere non valido, e converte le lettere minuscole in maiuscole. In questo modo l’IBAN viene accettato in qualsiasi formato il cliente lo inserisca.

Dopo la correzione, lo stesso identico IBAN con spazi viene accettato senza errori e il form avanza ai campi successivi.

Errore originale (log 3 aprile)

ValueError(code: 0): bcmod(): Argument #1 ($num1) is not well-formed at /var/www/html/nexum-smartsound/releases/20260320095511/app/Utilities/Iban.php:72 Causa: bcmod() riceve una stringa non numerica perché l'IBAN contiene spazi/trattini che non vengono rimossi prima della conversione lettere→numeri. Flusso: 1. Utente inserisce "IT 60 X054 2811 1010 0000 0123 456" 2. isValid() fa substr + str_replace (A→10, B→11, ...) 3. Spazi restano nella stringa → "18 29 60..." 4. bcmod("18 29 60...", "97") → BOOM: not well-formed

Riproduzione su staging

$ php artisan tinker >>> $iban = new App\Utilities\Iban('IT 60 X054 2811 1010 0000 0123 456'); >>> $iban->isValid(); ERROR: bcmod(): Argument #1 ($num1) is not well-formed Confermato: stesso identico errore dei log di produzione.

Fix applicato

File: app/Utilities/Iban.php PRIMA (riga 8-11): public function __construct( private ?string $iban = '', ) { } DOPO: public function __construct( private ?string $iban = '', ) { $this->iban = strtoupper(preg_replace('/[^A-Za-z0-9]/', '', $this->iban ?? '')); } Effetto: - Rimuove spazi, trattini, punti, qualsiasi non-alfanumerico - Converte minuscole → maiuscole - Gestisce null → stringa vuota - Il resto del codice (isValid, getAbi, getCab...) funziona invariato

Verifica post-fix

$ php artisan tinker (dopo il fix) >>> (new Iban('IT 60 X054 2811 1010 0000 0123 456'))->isValid() => true (spazi) >>> (new Iban('it60x0542811101000000123456'))->isValid() => true (minuscole) >>> (new Iban('IT60-X054-2811-1010-0000-0123-456'))->isValid() => true (trattini) >>> (new Iban(null))->isValid() => false (null → rifiutato, no crash) >>> (new Iban(''))->isValid() => false (vuoto → rifiutato, no crash) >>> (new Iban('IT60X0542811101000000123456'))->isValid() => true (formato corretto, nessuna regressione)
