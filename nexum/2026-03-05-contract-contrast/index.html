Riepilogo tecnico

Server: Cloudways Athena (staging)

File: resources/views/components/layouts/head-style.blade.php

Causa root: Il tema greyscale usa --color-green: #111 (quasi nero) come colore accent. I selettori CSS della paginazione usano var(--color-green) per il testo, risultando in testo nero su sfondo bianco (accettabile) e testo grigio su sfondo nero per la pagina attiva (illeggibile).

Specificità: Il selettore #app-body span (specificità alta) sovrascriveva il color: #fff di .page-link.active, forzando color: #3d4f5f (grigio scuro) su sfondo #111 (quasi nero).

Valori prima del fix:

/* Paginazione attiva — PRIMA */ .page-link.active: color=rgb(61, 79, 95) bg=rgb(17, 17, 17) ← ILLEGGIBILE .page-link normal: color=rgb(17, 17, 17) bg=rgb(255, 255, 255) ← ok ma al limite .page-link disabled: color=rgb(61, 79, 95) bg=transparent /* Bottone CSV — PRIMA */ btn-quiet: color=rgb(245, 245, 245) bg=rgb(77, 77, 77) icon (span): color=rgb(61, 79, 95) ← sovrascritta da #app-body span

Valori dopo il fix:

/* Paginazione attiva — DOPO */ .page-link.active: color=rgb(255, 255, 255) bg=rgb(17, 17, 17) ← BIANCO SU NERO ✓ .page-link normal: color=rgb(51, 51, 51) bg=rgb(255, 255, 255) ← SCURO SU BIANCO ✓ .page-link disabled: color=rgb(153, 153, 153) bg=transparent ← GRIGIO ✓ /* Bottone CSV — DOPO */ btn-quiet: color=rgb(245, 245, 245) bg=rgb(77, 77, 77) icon (span): color=rgb(245, 245, 245) ← eredita dal bottone ✓

Fix applicato:

/* Pagination active: force white text on dark bg */ .page-link.active, .active > .page-link, .page-item.active .page-link { color: #fff !important; } /* Pagination normal links: readable on white bg */ .page-link { color: #333 !important; } .page-link:hover { color: #111 !important; background-color: #eee; } /* Disabled pagination: lighter grey */ .page-item.disabled .page-link { color: #999 !important; } /* CSV button icon: inherit button color */ .btn.btn-quiet .material-symbols-outlined, .btn.btn-quiet .icon, .btn.btn-quiet span { color: inherit !important; }

Variabili CSS custom (da .env):

APP_COLOR_GREEN="#111" → --color-green (accent, quasi nero) APP_COLOR_GREEN_DEEP="#111" → --color-green-deep APP_COLOR_TEXT="#3d4f5f" → --color-text (grigio scuro) APP_COLOR_GREY non settato → --color-grey: #fafafa (default)

Nota: Il fix usa !important per vincere la specificità del selettore #app-body span. La soluzione definitiva sarebbe consolidare tutto in SCSS con specificità corretta (TODO esistente).
