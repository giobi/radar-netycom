✗
Il problema: il sito non si apriva

Aprendo la pagina di login, il sito mostrava una pagina bianca di errore (errore 500 — “errore interno del server”). Nessun utente poteva accedere.

La causa: durante un aggiornamento recente, un componente software necessario per l'ambiente di test (il “debugbar”, uno strumento che aiuta gli sviluppatori) non era stato installato. Il sito cercava di caricarlo, non lo trovava, e andava in errore.

✓
La soluzione: reinstallati i componenti

Abbiamo reinstallato tutti i pacchetti software necessari per l'ambiente di test. Il componente mancante è tornato al suo posto e il sito ha ripreso a funzionare immediatamente.

Server: Cloudways Athena (staging)

Path: /home/1174648.cloudwaysapps.com/estendonexumathena/public_html

URL: estendo.sel.re/auth/login

HTTP Status: 500 → 200

Errore primario:

Class "Barryvdh\Debugbar\ServiceProvider" not found at vendor/laravel/framework/src/Illuminate/Foundation/Application.php:880

Errore secondario (a cascata):

Call to undefined function Symfony\Component\ErrorHandler\ErrorRenderer\highlight_file() at vendor/symfony/error-handler/ErrorRenderer/HtmlErrorRenderer.php:251

Causa root: barryvdh/laravel-debugbar è in require-dev di composer.json. Un deploy (o un composer install --no-dev) ha rimosso il pacchetto dal vendor. Con APP_DEBUG=true, Laravel auto-discovers il ServiceProvider di Debugbar via package discovery e tenta di caricarlo → class not found → 500.

L'errore secondario (highlight_file()) si generava perché Symfony tentava di renderizzare l'errore HTML con syntax highlighting, ma highlight_file() è disabilitata nel php.ini di Cloudways → doppio crash.

Fix applicato:

ssh athena cd /home/1174648.cloudwaysapps.com/estendonexumathena/public_html composer install # Reinstalla tutte le dipendenze, incluse le dev (debugbar, ignition, ecc.)

Nota: Per evitare che si ripeta, il deploy su staging dovrebbe sempre usare composer install (con dev deps), non composer install --no-dev. In alternativa, spostare debugbar in un conditional register che controlla class_exists() prima di caricare il ServiceProvider.
